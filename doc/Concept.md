# Опис інструментів та їх призначення
## Характеристики
Опис основних характеристик кожного інструменту.

### 1 Minikube
- Підтримувані ОС та архітектури:

    Підтримує всі основні операційні системи, включаючи Windows, macOS і Linux. Він працює на архітектурах x86-64 та ARM, що робить його доступним для широкого кола користувачів.

- Можливість автоматизації:

    Підтримує інтеграцію зі скриптами та API, що дозволяє автоматизувати процеси розгортання та керування кластерами. Це корисно для розробників, які хочуть створювати автоматизовані середовища розробки та тестування.

- Наявність додаткових функцій:

    Пропонує базові інструменти для моніторингу та керування кластером, але його можливості в цьому напрямку обмежені в порівнянні з більш повнофункціональними рішеннями для виробничих середовищ.

### Kind (Kubernetes IN Docker)
- Підтримувані ОС та архітектури:

    Працює на Windows, macOS і Linux, підтримуючи архітектури x86-64 та ARM. Цей інструмент використовує Docker для створення кластерів, що робить його незалежним від основної ОС, але залежним від Docker.

- Можливість автоматизації:

    Добре інтегрується з CI/CD системами, що дозволяє легко автоматизувати розгортання кластерів для тестування. Цей інструмент особливо популярний серед розробників, які використовують CI/CD конвеєри для безперервної інтеграції та доставки.

- Наявність додаткових функцій:

    Не пропонує вбудованих інструментів для моніторингу або керування кластером, що робить його більш придатним для швидкого тестування та експериментів, ніж для постійного використання у виробничих середовищах.

### k3d
- Підтримувані ОС та архітектури:

    Підтримує Windows, macOS і Linux, працюючи на архітектурах x86-64 та ARM. Як і Kind, він використовує Docker для розгортання кластерів, що дозволяє легко розгортати Kubernetes у контейнерах.

- Можливість автоматизації:

    Легко інтегрується з CI/CD системами і забезпечує швидке розгортання та знищення кластерів, що робить його відмінним вибором для автоматизації процесів розробки та тестування.

- Наявність додаткових функцій:

    Не має розширених функцій для моніторингу та керування кластерами, але він забезпечує високу швидкість розгортання і легкість у використанні, що є його головними перевагами для локального тестування і PoC (proof of concept) проєктів.

# Висновки: 
Для стартапу AsciiArtify, кожен з розглянутих інструментів має свої переваги та недоліки.
minikube підійде для початкового знайомства та розробки, але може обмежувати при масштабуванні.
kind є добрим вибором для інтеграції з CI/CD системами та швидкого тестування, однак, має деякі обмеження щодо функціоналу Kubernetes.
k3d виглядає найкращим варіантом для PoC, оскільки дозволяє швидко створювати кластери, легкий у використанні та добре інтегрується з CI/CD системами.

Рекомендація: k3d для PoC та початкового розгортання, з можливістю переходу на більш масштабовані рішення в майбутньому.

# Кроки для відтворення тесту
У всіх трьох випадках сервіс розгортається за допомогою єдиного [шаблону](../hello-world.yaml)

## 1. minikube
### 1.1 Deploy:
```
curl -Lo minikube https://storage.googleapis.com/minikube/releases/latest/minikube-linux-amd64 && chmod +x minikube && ./minikube start
```
### 1.2 Тестовий сервіс:
```
k get all -A && k apply -f ./hello-world.yaml && k get all
k port-forward svc/hello-world-service 8081:8080 & 
```
### 1.3 Перевірка:
```
curl -v localhost:8081
curl localhost:8081
kubectl logs `kubectl get pods --output=name`
```
```
./minikube delete
```

## 2. kind 
### 2.1 Deploy:
```
[ $(uname -m) = x86_64 ] && curl -Lo ./kind https://kind.sigs.k8s.io/dl/v0.23.0/kind-linux-amd64 && chmod +x kind && ./kind create cluster --name asciiartify
```

### 2.2 Тестовий сервіс:
```
k get all -A && k apply -f ./hello-world.yaml && k get all
k port-forward svc/hello-world-service 8081:8080 & 
```
### 2.3 Перевірка:
```
curl -v localhost:8081
curl localhost:8081
kubectl logs `kubectl get pods --output=name`
```

## 3. k3d 
### 3.1 Deploy:
```
curl -s https://raw.githubusercontent.com/rancher/k3d/main/install.sh | bash && k3d cluster create k3d-asciiartify 
```

### 3.2 Тестовий сервіс:
```
k get all -A && k apply -f ./hello-world.yaml && k get all
k port-forward svc/hello-world-service 8081:8080 & 
```
### 3.3 Перевірка:
```
curl -v localhost:8081
curl localhost:8081
kubectl logs `kubectl get pods --output=name`
```
